# Release Checklist (Project-level)

Use this checklist for every production release. Assign an owner for each line when scheduling a release.

- [ ] Release name / number assigned
- [ ] Target release date/time and deployment window scheduled (Owner: Project Manager)
- [ ] Release owner identified (Owner: Project Manager)
- [ ] Change log / Release notes drafted (Owner: Product Manager)
- [ ] Acceptance criteria met for all included PRs (Owner: QA Lead / Product Manager)
- [ ] All PRs merged and CI green (Owner: Developers)
- [ ] Security scans completed and any critical findings mitigated (Owner: Security & Compliance Lead)
- [ ] Migration steps and rollback plan documented (Owner: DevOps / Tech Lead)
- [ ] Backups/snapshots taken (if applicable) (Owner: DevOps)
- [ ] Staging deployment completed and smoke tests passed (Owner: DevOps / QA Lead)
- [ ] Monitoring and alerting validated for new changes (Owner: DevOps / Data Analyst)
- [ ] Stakeholders notified of release schedule and impact (Owner: Project Manager)
- [ ] Support runbook and contact list updated (Owner: Project Manager / Product Manager)
- [ ] Post-deploy verification checklist executed (Owner: QA Lead / Developers)
- [ ] Post-release retrospective / action items scheduled (Owner: Project Manager)

Optional: attach relevant test run links, dashboards, and runbooks when marking items complete.
